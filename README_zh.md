## 💡 核心理念

> [!IMPORTANT]
> AI时代让造轮子变得触手可及，却也催生了无数同质化的重复项目。请冷静下来，停止浪费你的Token！

当灵感闪现时，不妨先去GitHub搜一搜，看看是否已经有契合你想法的开源项目。遇见相似项目，我们应该去参与和完善它，而不是重复制造仿品——除非你们的设计理念有着本质的不同。

---

## 🎯 这是什么？

NoReinventWheel 是一个 **AI 助手技能（Skill）**，旨在帮助开发者在开始新项目前，先搜索 GitHub 上已有的成熟开源方案，避免重复劳动，提高开发效率。

### 主要功能

| 功能 | 描述 |
|------|------|
| 🔍 **智能搜索** | 自动搜索 GitHub 上的相关开源项目 |
| 📊 **结果分析** | 评估项目的匹配度、活跃度、社区支持 |
| 💡 **决策建议** | 推荐复用、改造或自建的最佳路径 |
| 🌐 **双语支持** | 完整的中英文文档和输出 |

## 🚀 快速开始

### 安装

从 [Release](https://github.com/GDWhisper/NoReinventWheel.skill/releases) 下载最新版本的技能包：

| 版本 | 文件 |
|------|------|
| 🇨🇳 中文版 | `NoReinventWheel_zh.skill` |
| 🇺🇸 英文版 | `NoReinventWheel_en.skill` |

解压后将 `no-reinvent-wheel` 文件夹放入 Claude Skills 目录：

```bash
# Windows
C:\Users\<你的用户名>\.claude\skills\no-reinvent-wheel

# macOS/Linux
~/.claude/skills/no-reinvent-wheel
```

### 配置 GitHub Token（推荐）

> [!TIP]
> 为了提高 API 调用限制，建议配置个人访问令牌：

```bash
export GITHUB_TOKEN="ghp_your_token_here"
```

| 限制类型 | 请求次数/小时 |
|----------|---------------|
| ⬛ 未认证 | 60 |
| ✅ 已认证 | 5,000 |

## 📖 使用方法

### 基本使用

直接在 Claude 中表达开发意图即可触发搜索：

- "我想开发一个..."
- "我想做一个...项目"
- "有没有类似的..."
- 直接调用 `/no-reinvent-wheel` 或 `/wheel`

### 使用示例

> **输入：**
> ```
> 我想开发一个Windows缓存自动清理工具
> ```

> **输出：**
> ```
> GitHub 上已有成熟的类似项目，先给你汇报一下：
>
> 🐙 调研结果
>
> | 项目 | Stars | 语言 | 定位 |
> |------|-------|------|------|
> | BleachBit | 4.7k | Python | 成熟开源软件，Windows/Linux 全平台，功能最全 |
> | Winapp2 | 905 | VB.NET | 清洗规则数据库，为 CCleaner/BleachBit 提供扩展定义 |
> | WindowsCleanerUtility | 105 | Batch | 极简批处理脚本，清理临时文件 |
>
> 💡 我的判断
>
> BleachBit 已经是成熟开源标杆，如果你想做一个通用的"Windows 缓存清理工具"，
> 它基本覆盖了所有主流场景。
> ```

## 🤖 AI 集成与工作原理

### 触发场景

当用户表达以下意图时，自动触发搜索：

- "我想做一个..."
- "我要开发..."
- "有没有类似的..."
- 直接调用 `/no-reinvent-wheel` 或 `/wheel`

### 强触发词（立即拦截）

<details>
<summary>看到这些词时，代表用户明确表达了"我要从零造一个完整的东西"的意图：</summary>

#### "从零/手写/自己实现"系列
- "帮我**从零**写一个……"
- "不用第三方库，**纯手写**一个……"
- "我不想依赖外部包，**自己实现**一个……"

#### "项目/模块/系统"系列
- "帮我创建一个**项目**，实现……"
- "写一个完整的**模块**，用于……"
- "设计一个**系统**，能够……"

#### "我想做/准备做"系列
- "我有个点子，我想做一个……"
- "我准备开发一个……"
</details>

### 高频轮子重灾区

用户的指令中同时包含"写代码的动作" + "以下高频轮子领域"时触发：

| 类别 | 关键词 |
|------|--------|
| **认证权限类** | 登录/注册/鉴权/JWT/权限管理/OAuth |
| **数据解析类** | 爬虫/解析器/导出Excel/读取PDF/解析XML |
| **基础基建类** | 聊天室/WebSocket/定时任务/队列/日志系统 |
| **常见业务类** | 购物车/支付接口/分页组件/富文本编辑器/文件上传 |
| **算法工具类** | 日历控件/倒计时/加密解密/图片压缩 |

### 不触发的场景

以下情况请不要触发（正常回答）：

- ✅ 用户明确说"为了学习/教学"
- ✅ 用户只是在问一个函数的写法和参数
- ✅ 用户在调试/修复现有代码的 Bug
- ✅ 用户问的是正则、具体算法逻辑等非常细粒度的问题
- ✅ 用户的"写"是在现有代码基础上改，而不是从零生成

### 工作流程

```
graph TD
    A[用户提出需求] --> B{识别意图}
    B -->|需要完整功能| C[触发搜索]
    B -->|学习/调试/细粒度| D[直接回答]
    C --> E[执行 GitHub 搜索]
    E --> F[分析搜索结果]
    F --> G{匹配度评估}
    G -->|✅ 完全匹配| H[建议直接使用/Fork]
    G -->|🔧 部分匹配| I[建议基于此改造]
    G -->|📝 参考价值| J[仅作学习参考]
    G -->|❌ 无合适项目| K[确认是否自建]
    H --> L[结束]
    I --> L
    J --> L
    K --> L
```

### 输出格式

脚本返回统一的 JSON 结构：

```json
{
  "lang": "zh",
  "status": "ok",
  "query": "番茄钟",
  "total_count": 1234,
  "returned_count": 10,
  "items": [
    {
      "rank": 1,
      "full_name": "user/repo",
      "description": "A pomodoro timer app",
      "url": "https://github.com/user/repo",
      "stars": 12300,
      "forks": 1200,
      "language": "TypeScript",
      "pushed_days_ago": 3,
      "created_at": "2024-01-01",
      "topics": ["productivity", "timer"],
      "license": "MIT"
    }
  ]
}
```

**AI 展示规则：**

| 字段 | 规则 |
|------|------|
| `lang: "en"` | AI 负责翻译成用户语言再展示 |
| `pushed_days_ago: 3` | AI 转换成 "3天前" 或 "3 days ago" |
| `stars: 12300` | AI 格式化 "12.3k" |
| `status: "error"` | AI 根据 `code` 和 `message` 生成友好的错误提示 |

## 🌟 为什么选择 NoReinventWheel？
