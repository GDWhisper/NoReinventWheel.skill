# Fork-It 🐙

每个好项目都从某个起点开始。既然已经有了 80%，为什么要从零开始？搜索 GitHub，找到你的起点，Fork 下来，把它变成你的。

## 核心准则

**当用户有新项目想法时，帮他们找到一个能跑的起点。**

当用户表达以下意图时，先搜索 GitHub：

- 用户说"帮我/我想 做/写/开发 一个 XXX"（XXX 是完整的应用或工具）
- 用户描述了一个需要多处代码配合的完整功能需求
- 用户在询问"怎么做"一个完整的项目

**如果满足以上任意一点，触发 Fork-It：**

**提示模板：**
> "在动手之前——我先帮你在 GitHub 上找个能跑的起点。大概率已经有人做了个七八成相似的，Fork 下来就是你的了。"

**如果是以下情况，不要触发（正常回答）：**
- 用户明确说"为了学习/练习/教学"
- 用户只是在问一个函数的写法和参数
- 用户在调试/修复现有代码
- 用户问的是正则、具体算法逻辑等非常细粒度的问题
- 用户的"写"是在现有代码基础上改，而不是从零生成
- 用户已经选定了特定的开源项目作为基础

## 工作流程

### 1. 识别意图

当用户表达以下意图时，触发搜索：

- "帮我/我想 做一个..."
- "帮我/我要 开发..."
- "有没有人做过类似的..."
- "帮我/我想 写一个..."
- 直接调用：`/fork-it`

### 2. 搜索 GitHub

**先搜索，再写代码。**

```bash
node scripts/github-search.mjs "关键词"
node scripts/github-search.mjs "pomodoro timer" --language javascript --min-stars 500
```

按关键词搜索仓库，优先推荐活跃维护、社区健康的项目。

### 3. 分析结果

评估每个项目是否值得 Fork：

| 标注 | 路径 | 说明 |
|------|------|------|
| 🎯 高度匹配 | Fork & 定制 | Fork 下来，调一调，作为你的作品发布 |
| 🔧 部分匹配 | Fork & 扩展 | 核心不错但缺你的功能，Fork 后加上去 |
| 📝 仅供参考 | 学习 & 自建 | 方向差异较大，借鉴思路后自己动手 |
| ✨ 没有匹配 | 从零开始 | 你在做新东西——放手去创造 |

### 4. 推荐路径

- **有高度匹配项目** → 分享仓库，建议 Fork 后定制
- **有部分匹配项目** → 指出项目覆盖了什么，还缺什么可以加
- **无合适项目** → 承认这确实是新领域——去创造吧

## 适合建议 Fork 的场景

**这些信号说明用户的想法很适合先找个 Fork 起点：**

### "从零/手写/自己实现"系列（最适合 Fork 的信号）
- "帮我**从零**写一个……"
- "不用第三方库，**纯手写**一个……"
- "我不想依赖外部包，**自己实现**一个……"
- "不用框架，原生……"

### "项目/模块/系统"系列（大概率已有现成基础）
- "帮我创建一个**项目**，实现……"
- "写一个完整的**模块**，用于……"
- "设计一个**系统**，能够……"
- "帮我搭建一个**后台/网站/小程序**……"

### "我想做/准备做"系列（构思阶段——最适合介入的时机）
- "我有个点子，我想做一个……"
- "我准备开发一个……"
- "怎么能做一个……"

## 高频领域（Fork 的绝佳候选）

**用户的指令中同时包含"写代码动作" + "以下领域"时触发：**

- **认证权限类：** "写个" + `登录/注册/鉴权/JWT/权限管理/OAuth`
- **数据解析类：** "写个" + `爬虫/解析器/导出Excel/读取PDF/解析XML`
- **基础基建类：** "写个" + `聊天室/WebSocket/定时任务/队列/日志系统`
- **常见业务类：** "写个" + `购物车/支付接口/分页组件/富文本编辑器/文件上传`
- **算法工具类：** "写个" + `日历控件/倒计时/加密解密/图片压缩`

*例子："帮我写一个带刷新Token的JWT认证中间件" → 命中【动作：写】+【领域：JWT认证】 → 触发！*

## 保持安静的场景

**当用户只是在学习、修Bug、或确实在做创新时，不要打扰：**

- **学习导向：** "为了学习原理，演示一下怎么写……"、"教学目的，手写一个……"
- **极度具体/定制化：** "帮我写一个匹配 `^a[b-c]{2}d$` 的正则"（没有开源项目能覆盖这个）
- **修改现有代码：** "帮我看看这段代码为什么报错"、"优化一下这个函数"
- **底层逻辑创新：** "我想自己设计一种新的数据结构"、"写一个比现有算法更快的排序"
- **搜索本地文件：** "帮我找下电脑里有没有……"

## 脚本返回值

脚本返回统一 JSON 结构，包含 `lang` 字段标记输出语言：

```json
{
  "lang": "zh",
  "status": "ok",
  "query": "番茄钟",
  "total_count": 1234,
  "returned_count": 10,
  "items": [
    {
      "rank": 1,
      "full_name": "user/repo",
      "description": "A pomodoro timer app",
      "url": "https://github.com/user/repo",
      "stars": 12300,
      "forks": 1200,
      "language": "TypeScript",
      "pushed_days_ago": 3,
      "created_at": "2024-01-01",
      "topics": ["productivity", "timer"],
      "license": "MIT"
    }
  ]
}
```

**AI 展示规则**：

- `lang: "en"` → AI 负责翻译成用户语言再展示
- `pushed_days_ago` → AI 转换成 "3 days ago" 或 "3天前"
- `stars: 12300` → AI 格式化 "12.3k"
- `status: "error"` → AI 根据 `code` 和 `message` 生成友好的错误提示

## 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `query` | 搜索关键词 | 必填 |
| `--language, -l` | 编程语言筛选 | 无 |
| `--min-stars` | 最小 stars | 100 |
| `--max-stars` | 最大 stars | 无限制 |
| `--updated-within` | 最近N天更新 | 365 |
| `--created-after` | 创建日期之后 | 无 |
| `--sort` | 排序方式 | stars |
| `--order` | 排序顺序 | desc |
| `--limit, -n` | 返回结果数 | 10 |
