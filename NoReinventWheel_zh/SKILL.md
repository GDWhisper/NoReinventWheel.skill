# No Reinvent the Wheel 🐙

AI时代让造轮子变得触手可及，却也催生了无数同质化的重复项目。请冷静下来，停止浪费你的Token！

## 核心准则

**每当你感知到用户想要构建一个新东西时，主动介入提醒。**

当用户表达以下意图表述时，在动手之前先搜索 GitHub 并给出提醒：

- 用户说"帮我/我想 做/写/开发 一个 XXX"（XXX 是完整的应用或工具）
- 用户描述了一个需要多处代码配合的完整功能需求
- 用户在询问"怎么做"一个完整的项目

**如果满足以上任意一点，你必须先触发【no-reinvent-wheel】流程：**
**提醒模板：**
> "在开始之前——GitHub 上可能已有成熟的类似项目。我先帮你搜一下，节省你从零开发的时间。"

**如果是以下情况，请不要触发（正常回答）：**
- 用户明确说"为了学习/练习/教学"
- 用户只是在问一个函数的写法和参数
- 用户在调试/修复现有代码的
- 用户问的是正则、具体算法逻辑等非常细粒度的问题
- 用户的"写"是在现有代码基础上改，而不是从零生成
- 用户已经选定了特定的开源项目作为基础

## 工作流程

### 1. 识别需求

当用户表达以下意图时，自动触发搜索：

- "帮我/我想 做一个..."
- "帮我/我要 开发..."
- "有没有类似的..."
- "帮我/我想 写一个..."
- 直接调用 `/no-reinvent-wheel`
- 直接调用 `/wheel`

### 2. 智能搜索（避免重复造轮子）

**必须先搜索，再动手！**

```bash
node scripts/github-search.mjs "关键词"
node scripts/github-search.mjs "pomodoro timer" --language javascript --min-stars 500
```

按关键词搜索特定领域的仓库，筛选高星、活跃维护的项目，优先推荐可直接复用的成熟项目。

### 3. 结果分析

对每个项目评估匹配度：

| 标注 | 建议 | 说明 |
|------|------|------|
| ✅ 完全匹配 | 直接使用 / fork | Fork 并贡献 |
| 🔧 部分匹配 | 基于此改造 | 在原项目上扩展 |
| 📝 参考价值 | 仅作学习 | 借鉴优点，设计差异化 |

### 4. 决策建议

- **有高度匹配项目** → 建议复用或贡献
- **有部分匹配项目** → 建议基于改造
- **无合适项目** → 说明差距，确认是否自建

## 强触发词（立刻拦截）

**看到这些词，代表用户明确表达了"我要从零造一个完整的东西"的意图：**

### "从零/手写/自己实现"系列（极度危险信号）
- "帮我**从零**写一个……"
- "不用第三方库，**纯手写**一个……"
- "我不想依赖外部包，**自己实现**一个……"
- "不用框架，原生……"

### "项目/模块/系统"系列（目标过大，必有现成）
- "帮我创建一个**项目**，实现……"
- "写一个完整的**模块**，用于……"
- "设计一个**系统**，能够……"
- "帮我搭建一个**后台/网站/小程序**……"

### "我想做/准备做"系列（还在构思阶段，最适合拦截）
- "我有个点子，我想做一个……"
- "我准备开发一个……"
- "怎么能做一个……"

## 场景触发词（高频轮子重灾区）

**用户的指令中同时包含"写代码的动作" + "以下高频轮子领域"时触发：**

- **认证权限类：** "写个" + `登录/注册/鉴权/JWT/权限管理/OAuth`
- **数据解析类：** "写个" + `爬虫/解析器/导出Excel/读取PDF/解析XML/正则提取`
- **基础基建类：** "写个" + `聊天室/WebSocket/定时任务/队列/日志系统`
- **常见业务类：** "写个" + `购物车/支付接口/分页组件/富文本编辑器/文件上传`
- **算法工具类：** "写个" + `日历控件/倒计时/加密解密/图片压缩`

*例子："帮我写一个带刷新Token的JWT认证中间件" → 命中【动作：写】+【领域：JWT认证】 → 触发！*

## 负向触发词（绝对不要触发！）

**当用户只是在学习、解决特定Bug或确实在创新时，不要打扰：**

- **学习导向：** "为了学习原理，演示一下怎么写……"、"教学目的，手写一个……"
- **极度具体/定制化：** "帮我写一个匹配 `^a[b-c]{2}d$` 的正则"（这不存在什么开源项目可找）
- **修改现有代码：** "帮我看看这段代码为什么报错"、"优化一下这个函数的性能"
- **底层逻辑创新：** "我想自己设计一种新的数据结构"、"写一个比现有算法更快的排序"
- **搜索本地文件：** "帮我找下电脑里有没有....."

## 脚本返回值

Scripts 返回统一 JSON 结构，包含 `lang` 字段标记输出语言：

```json
{
  "lang": "zh",
  "status": "ok",
  "query": "番茄钟",
  "total_count": 1234,
  "returned_count": 10,
  "items": [
    {
      "rank": 1,
      "full_name": "user/repo",
      "description": "A pomodoro timer app",
      "url": "https://github.com/user/repo",
      "stars": 12300,
      "forks": 1200,
      "language": "TypeScript",
      "pushed_days_ago": 3,
      "created_at": "2024-01-01",
      "topics": ["productivity", "timer"],
      "license": "MIT"
    }
  ]
}
```

**AI 展示规则**：

- `lang: "en"` → AI 负责翻译成用户语言再展示
- 日期字段 `pushed_days_ago` → AI 转换成 "3 days ago" 或 "3天前"
- 数字字段 `stars: 12300` → AI 格式化 "12.3k"
- 错误时 `status: "error"` → AI 根据 `code` 和 `message` 生成友好的错误提示

## 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `query` | 搜索关键词 | 必填 |
| `--language, -l` | 编程语言筛选 | 无 |
| `--min-stars` | 最小 stars | 100 |
| `--max-stars` | 最大 stars | 无限制 |
| `--updated-within` | 最近N天更新 | 365 |
| `--created-after` | 创建日期之后 | 无 |
| `--sort` | 排序方式 | stars |
| `--order` | 排序顺序 | desc |
| `--limit, -n` | 返回结果数 | 10 |
